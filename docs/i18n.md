# Internationalization

The app supports multiple locales and uses JSON-based key/value localization.

## Locale files

Frontend locale files are stored under `src/i18n/locales/`:

- `en.json`
- `zh.json`
- `ja.json`

Backend prompt templates are also language-aware under:

- `server/src/utils/prompt_templates/`

## Key management rules

- Add new user-facing strings to locale files, not inline literals.
- Keep key names stable and namespaced (for example `common.*`, `storyActions.*`).
- Avoid creating near-duplicate keys with minor wording differences.
- Keep interpolation placeholders consistent across locales.

## Consistency workflow

1. Add or update key in all required locale files.
2. Verify UI usage through existing i18n hook/util patterns.
3. Run `npm run lint:i18n`.
4. Verify key rendering in the UI paths you changed.

## Introducing a new locale

1. Add a new locale JSON file under `src/i18n/locales/`.
2. Follow existing key structure and namespace conventions.
3. Ensure fallback behavior is well-defined in i18n setup.
4. Update tests or snapshots that rely on locale-dependent strings.

## Backend language implications

- Prompt templates and language preferences influence generated AI content.
- If you add language-dependent backend behavior, validate both template loading and app settings persistence.
